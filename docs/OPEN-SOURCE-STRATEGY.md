# Open Source Strategy

## First objective

The first public objective is credibility. HBN should be understandable and technically modest enough that outside contributors can evaluate it without reading hype into it.

## Strategy

- publish the protocol definition early
- keep runtime scope narrow and inspectable
- document governance from the start
- accept external review as part of protocol formation
- avoid premature complexity that would obscure the core language

## Release posture

This repository should be maintained as:

- public by default
- review-oriented
- explicit about limits
- stable enough for discussion, not over-claimed as finished infrastructure

## Public domain posture

The public site strategy should remain simple:

- `usehbn.org` as canonical public site
- `usehbn.com` as permanent redirect and semantic alias

This keeps documentation, semantics, and public distribution aligned.
