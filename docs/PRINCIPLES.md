# HBN Principles

The HBN protocol is guided by the following principles:

1. Human is the root of trust.
2. No hidden behavior is allowed.
3. Intent should be explicit before execution.
4. Constraints should be visible and validated.
5. Risk should be named, not implied.
6. Claims should be proportionate to evidence.
7. Every change should be traceable in code, docs, or logs.
8. Simplicity is preferred over unnecessary complexity.

These principles apply to code, documentation, agent behavior, and future system design.
