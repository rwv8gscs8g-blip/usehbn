# Founding Notes

HBN begins from a practical observation: AI-assisted development is useful, but too often under-governed. Teams can produce output faster than they can inspect intent, assumptions, or risk.

The first HBN release therefore focuses on protocol fundamentals:

- explicit activation
- structured intent
- contribution consent
- truth checks
- guardian warnings
- governance

This repository is the first public artifact, not the final form of the protocol. Its purpose is to create a disciplined starting point that can evolve in public.
