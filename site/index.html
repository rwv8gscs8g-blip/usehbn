HBN — Human Brain Net

HBN is an open protocol for safe, structured, and evolvable AI-assisted software engineering.

Built as a protocol first, HBN defines how AI-assisted software work should be activated, structured, validated, and documented under human authority.

Read the repository Read the protocol

What It Is

HBN is not a product or framework. It is a protocol layer for structured AI-assisted engineering: explicit activation, structured intent, visible validation, and documented outcomes.

Why It Exists

AI-assisted work often moves faster than teams can inspect intent, constraints, or confidence. HBN exists to keep human control, traceability, and safety language visible.

How It Works

  1. Activate with usehbn or use hbn.
  2. Structure the request into objective, constraints, risks, and validation requirements.
  3. Run truth barrier and guardian checks.
  4. Persist execution logs and state records under human authority.
  5. Return structured, explainable output under human authority.

Core Principles

  • Human is the root of trust.
  • No hidden behavior is allowed.
  • Every change must be traceable.
  • Validation should not be bypassed.
  • Clarity is preferred over complexity.

Real Execution

The current HBN scaffold is executable. The CLI parses input, applies validation rules, writes a JSON execution log, and updates a JSON state layer with execution history, decisions, and context.
