# Governance

## Founder

HBN was created by Luis Mauricio Junqueira Zanin.

The founder is the initial steward of the official protocol definition until maintainership is expanded through public, review-based governance.

## Root Of Trust

The human operator, reviewer, or maintainer is the root of trust in HBN.

No automated output, generated claim, or inferred behavior overrides human judgment, human review, or human accountability.

## Core Rules

1. HBN must remain an open protocol.
2. The official protocol definition in this repository cannot be privatized.
3. Safety language must remain specific, testable, and non-exaggerated.
4. Safety mechanisms must not be weakened in the name of speed or convenience.
5. Documentation must not claim capabilities that the implementation does not provide.

## Change Control

Protocol-affecting changes require structured validation:

- implementation review when code changes
- documentation updates when semantics change
- tests when behavior changes
- changelog visibility for release-relevant changes

Changes to activation, intent structure, consent behavior, truth barrier rules, guardian behavior, governance, or licensing are protocol-sensitive by default.

## Public Integrity

HBN is open to contribution, but the protocol does not evolve by informal drift. Official changes require visible review and a public record in the repository.
